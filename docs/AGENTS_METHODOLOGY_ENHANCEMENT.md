# TZUKWAN 智能体方法论增强方案

> 基于 "科研大喵" 论文写作方法论资料的结构化提炼与智能体融合方案

## 一、方法论体系总览

### 1.1 研究创新五层次模型

```
┌────────────────────────────────────────────────────────────┐
│ L5 构建型 │ 从海量数据探索→提出全新理论（高风险高回报）        │
├────────────────────────────────────────────────────────────┤
│ L4 比较型 │ 多理论同领域对比→选出最优解释框架                │
├────────────────────────────────────────────────────────────┤
│ L3 整合型 │ 多理论有机融合→形成复合跨领域框架                │
├────────────────────────────────────────────────────────────┤
│ L2 延伸型 │ 核心理论+新因素→延伸验证理论适用边界            │
├────────────────────────────────────────────────────────────┤
│ L1 应用型 │ 成熟理论→新兴领域（新手友好）                    │
└────────────────────────────────────────────────────────────┘
```

### 1.2 Research Gap 识别矩阵

| Gap类型 | 表现特征 | 文献烘托策略 | 创新切入点 |
|---------|----------|--------------|-----------|
| **旧** | 研究老化固定，未考虑新兴现象/方法/理论 | 用陈旧文献烘托新意 | 新现象研究、新方法探索、新理论验证 |
| **窄** | 研究偏颇失衡，过于乐观/狭窄/极端 | 用互补/矛盾文献烘托 | 补充平衡视角、多面性探索、调和争议 |
| **泛** | 研究杂乱分散，不同学科自说自话 | 用散落多领域文献烘托 | 跨学科整合、统一框架、构建脉络 |
| **少** | 某现象缺乏研究或有效分析方法 | 划归成熟领域+借鉴方法 | 开创新领域、建立新框架 |

### 1.3 研究设计三思路

| 维度 | 探索式(Inductive) | 验证式(Deductive) | 结合式(Mixed) |
|------|------------------|------------------|---------------|
| **起点** | 数据/现象 | 理论 | 理论大框架+数据探索 |
| **流程** | 数据→提炼规律→形成理论 | 理论→假设→验证 | 理论指导+数据丰富理论 |
| **分析** | Content/Narrative/Thematic | SEM/回归/假设检验 | 定性与定量结合 |
| **适用** | 新兴领域/理论空白 | 理论成熟/假设清晰 | 复杂问题/全面理解 |

### 1.4 文献阅读二八定律

```
80% 筛选粗读阶段                    20% 精读巩固阶段
├── 课题探索 (Literature Exploration)   ├── 文献整理 (Literature Analysis)
├── 文献定位 (Literature Mapping)       ├── 写作运用 (Literature Internalisation)
└── 粗读筛选 (Literature Screening)     └── 温故知新 (持续迭代)
```

### 1.5 论文结构经典模式

| 模式 | 核心逻辑 | 代表结构 | 适用场景 |
|------|----------|----------|----------|
| **理论应用型** | 成熟理论→新领域应用 | 现状→Gap→理论应用→数据验证→讨论 | 新手入门 |
| **理论延伸型** | 核心理论+扩展因素 | 基础模型→延伸框架→验证→对比讨论 | 理论深化 |
| **理论整合型** | 多理论融合 | 模型梳理→对比→整合框架→验证 | 跨学科研究 |
| **理论对比型** | 多理论同场竞技 | 多模型介绍→同数据测试→对比分析 | 方法论文 |
| **理论构建型** | 从数据归纳理论 | 数据探索→模式识别→理论提出→验证 | 开创性研究 |

## 二、各智能体方法论增强详情

### 2.1 🎓 导师智能体 Dr. Mentor (战略统筹层)

#### 新增方法论框架

**科研六步工作法**
```
1. 观察现象(提出问题) 
   → 2. 阅读文献(理论解释) 
   → 3. 提出假设(明确变量)
   → 4. 概念模型(作出预测) 
   → 5. 数据收集(测量变量) 
   → 6. 数据分析(验证假设)
```

**五大思维模式**
- **正向思维**：方向确定→文献阅读→数据收集→数据分析→论文写作→定稿投出
- **逆向思维**：从目标期刊出发→确定研究方向→筛选文献→收集数据
- **往复思维**：循环往复调整，遇到具体问题回归文献寻找答案
- **全局思维**：讲一个好故事，确保各章节逻辑连贯
- **延伸思维**：从当前研究延伸到下一个课题

**协调工作流**
```
用户提出需求
    ↓
Dr. Mentor 评估任务复杂度
    ↓
├─ 需要文献调研 → 调用 Dr. Lit (二八定律筛选)
├─ 需要选题评估 → 调用 Dr. Topic (Gap识别矩阵)
├─ 需要实验设计 → 调用 Dr. Lab (三思路选择)
├─ 需要论文写作 → 调用 Dr. Write (经典结构选择)
├─ 需要内部评审 → 调用 Dr. Peer (审稿人视角)
└─ 需要基金申报 → 调用 Dr. Grant (申报策略)
    ↓
整合所有反馈 → 给出战略建议
```

### 2.2 📚 文献智能体 Dr. Lit (文献层)

#### 新增方法论：系统性文献综述方法

**文献搜索五技巧**
1. **逻辑词组合**：AND/OR/AND NOT 精准筛选
2. **模糊搜索**：*通配符扩大范围 (如 Consumer*)
3. **固定短语**："exact phrase" 精准定位
4. **顺序限定**：pre/n 指定词序位置
5. **范围限定**：w/n 指定词间距离

**文献筛选三边界**
```
核心文献 ←────── 延伸文献 ──────→ 零散文献
(直接相关)       (间接相关)        (参考价值)
   ↓                ↓                 ↓
精读分析         粗读了解           选择性阅读
```

**文献阅读五阶段**
| 阶段 | 英文 | 核心任务 | 产出 |
|------|------|----------|------|
| 探索 | Exploration | 拆解课题亮点，制定关键词 | 课题理解 |
| 定位 | Mapping | 多数据库搜索，粗读分析 | 文献清单 |
| 筛选 | Screening | 评估主题相关性，分类归档 | 精选文献 |
| 整理 | Analysis | 精读归纳，关联课题 | 文献笔记 |
| 巩固 | Internalisation | 标记引用，温故知新 | 写作素材 |

**综述类型选择指南**
```
研究问题清晰可量化 ──→ Systematic Review (系统性综述)
     ↓
问题宽泛跨领域 ──────→ Semi-systematic Review (半系统性综述)
     ↓
需提出新理论框架 ────→ Integrative Review (整合性综述)
```

### 2.3 🎯 选题智能体 Dr. Topic (创新层)

#### 新增方法论：创新点挖掘体系

**创新点评估矩阵 (44条思路)**

从三大角度挖掘创新点：
- **现象角度**：新兴现象/人群/趋势/过程/规律/对比
- **理论角度**：经典理论验证/新理论提出/理论融合/对比/延伸
- **方法角度**：新方法提出/对比/适用范围/改进/混合

**创新深度选择器**
```
文献基础  稀少 ──────────────────────── 丰富
     │                                  │
Gap类型   泛(杂乱)  少(空白)  旧(老化)  窄(偏颇)
     │       │        │        │        │
创新策略  整合    开创新领域  应用/延伸  补充/平衡
     │       │        │        │        │
创新层次   L3       L5       L1/L2    L2/L4
```

**选题可行性五维评估**
1. 学术贡献度 (Academic Contribution)
2. 方法论创新性 (Methodological Innovation)
3. 数据可及性 (Data Accessibility)
4. 时间可行性 (Temporal Feasibility)
5. 发表潜力 (Publication Potential)

### 2.4 🔬 实验智能体 Dr. Lab (方法层)

#### 新增方法论：研究设计与数据分析

**研究设计选择决策树**
```
有无既定理论指导？
    ├─ 无 → 探索式研究 (定性主导)
    │         └── Content/Narrative/Thematic分析
    ├─ 有 → 验证式研究 (定量主导)
    │         └── 多元回归/结构方程/假设检验
    └─ 部分有 → 结合式研究 (混合方法)
              └── 理论框架+数据探索相互丰富
```

**理论模型构建方法**

*横向拓展* (增加解释维度)
```
基础模型 → 多因素模型 → 多角度模型 → 主次划分模型
(单因单果)   (多因单果)     (辩证视角)     (分层解释)
```

*纵向深挖* (揭示作用机制)
```
直接效应 → 中介模型 → 调节模型 → 有调节的中介/多层级模型
(X→Y)      (X→M→Y)    (X→Y受W调节)   (复杂机制)
```

**数据分析方法选择矩阵**

| 因变量 | 自变量 | 关系 | 方法 |
|--------|--------|------|------|
| 定量 | 定量(1个) | 简单 | 简单回归 |
| 定量 | 定量(多个) | 复杂 | 多元回归/MANOVA |
| 定量 | 定性 | 复杂 | 方差分析/典型相关 |
| 非定量 | 混合 | 复杂 | 逻辑回归/判别分析 |
| 不明确 | 混合 | 未知 | 因子分析/聚类分析 |
| 不明确 | 不明确 | 结构关系 | 结构方程模型(SEM) |

**数据缺失处理策略**
```
缺失程度评估
    ↓
<10% → 删除缺失cases (Listwise/Pairwise)
    ↓
10-15% → 评估是否随机
    ├─ 随机 → Complete case / Mean substitution
    └─ 非随机 → Modelling-based approach
```

### 2.5 ✍️ 写作智能体 Dr. Write (写作层)

#### 新增方法论：论文结构与故事线

**论文时间分配建议**
```
总时长: 100%
├── 前期准备: 55-60%
│   ├── 文献阅读筛选: 50%
│   └── 精读整理: 5-10%
├── 实验/数据分析: 20-30%
└── 论文撰写: 15-20%
    ├── 初稿写作: 10-15%
    └── 修改调整: 3-5%
```

**IMRaD 结构强化版**

| 章节 | 五大核心问题 | 方法论要点 |
|------|-------------|-----------|
| **Introduction** | What? Why? | Research Gap定位、创新点呈现 |
| **Literature Review** | 前人做了什么？ | 文献脉络梳理、Gap论证 |
| **Method** | How? | 研究设计选择、数据收集、分析方法 |
| **Results** | 发现了什么？ | 图表结合、客观呈现 |
| **Discussion** | So what? | 理论解释、与前人对比、局限性 |
| **Conclusion** | 贡献与展望 | 理论贡献、实践意义、未来方向 |

**八种故事线模式**
1. **填补空白型**：Gap A存在→本文填补→贡献
2. **矛盾调和型**：文献A vs 文献B矛盾→本文解释→贡献
3. **深化拓展型**：文献A研究浅→本文深挖→贡献
4. **跨域应用型**：理论A在领域B未用→本文应用→贡献
5. **方法改进型**：现有方法有缺陷→本文改进→贡献
6. **综合比较型**：多方法对比→本文最优→贡献
7. **全新构建型**：新现象→探索规律→新建理论
8. **反直觉型**：常识认为X→本文发现非X→解释

**理论渗透各章节指南**

| 章节 | 定性研究 | 定量研究 |
|------|----------|----------|
| Intro | 理论作为理解现象的视角 | 理论作为变量关系来源 |
| LR | 展示理论情境多样性 | 汇总主流变量与假设路径 |
| Framework | 理论作为"视角"构建概念框架 | 理论构建变量结构模型 |
| Method | 用于设计问题提纲 | 用于指导变量与问卷设计 |
| Discussion | 理论解释参与者行为逻辑 | 理论解释路径是否被支持 |
| Conclusion | 理论理解与情境化 | 总结理论路径验证 |

### 2.6 🔍 审稿智能体 Dr. Peer (评审层)

#### 新增方法论：审稿流程与回复策略

**审稿九状态流程**
```
Submitted → Under editorial check → With editor 
    → Associate editor assigned → Reviewer invited 
    → Under review → Required reviews completed 
    → Decision in progress → Revise/Completed
```

**审稿结果应对策略**

| 结果类型 | 概率 | 应对策略 |
|----------|------|----------|
| Accept | <1% | 直接庆祝 |
| Minor Revision | 常见 | 逐条回复，快速修改 |
| Major Revision | 常见 | 认真修改每条意见，补充实验 |
| Reject & Resubmit | 中等 | 按大修标准修改后重投 |
| Transfer | 中等 | 评估转投期刊，可选择接受或改投 |
| Reject | 常见 | 分析原因，改投他刊 |

**回复审稿人六策略**

| 情况 | 策略 | 操作方式 |
|------|------|----------|
| 同意&易改 | 二话不说改 | 直接修改，礼貌感谢 |
| 同意&难改 | 且改且珍惜 | 认真修改，说明困难 |
| 同意&不改 | Limitation法 | 放Limitation讨论 |
| 不同意&易改 | 放弃自我 | 满足审稿人，说明妥协 |
| 不同意&难改 | 求同存异 | 部分接受，保留倔强 |
| 不同意&不改 | 有理开怼 | 有理有节，不卑不亢 |

**审稿人心理分析**
- 第一印象：标题、摘要、创新点
- 认知负荷：多用小标题、要点列举、框图
- 可信度：引用最新高质量文献
- 风险感知：主动识别难点并给预案
- 学术品位：用词精准，避免夸大

### 2.7 💰 基金智能体 Dr. Grant (申报层)

#### 新增方法论：基金申报策略

**基金类型选择**

| 类型 | 重点 | 创新点要求 | 适合阶段 |
|------|------|-----------|----------|
| 青年基金 | 潜力与成长性 | 2-3个，有文献支撑 | 早期研究者 |
| 面上项目 | 基础与可行性 | 实质性创新 | 有积累的研究者 |
| 重点项目 | 重大问题突破 | 突破性创新 | 知名学者 |

**申报书结构方法论**

```
1. 研究背景与意义
   └── 宏观→微观：国家战略→领域问题→科学问题
   
2. 国内外研究现状
   └── 按主题/方法/时间线梳理→明确Gap
   
3. 研究目标与内容
   └── 目标具体可量化→内容分解为3-5个问题
   
4. 研究方案
   └── 技术路线图→操作步骤→难点预案
   
5. 创新点 (灵魂)
   └── 2-3个→每个首句明确创新→说明与现有区别
   
6. 预期成果
   └── 具体可量化：论文(级别+数量)、专利、数据库
   
7. 研究基础
   └── 直接相关成果→数据基础→合作基础
```

**创新点写作公式**
```
首句：明确创新所在
后续句：说明与现有工作的区别
引用：用文献证明"前人未做"
```

### 2.9 ⚖️ 真实性审查官 Dr. Integrity (质控层)

#### 方法论融入：真实科研全流程质控

**审查要点**
1. **实验真实性**：具体到可操作的方案、时间地点记录
2. **数据真实性**：可追溯来源、样本量合理、分布正常
3. **结论支撑度**：每个核心结论有对应数据分析
4. **引用真实性**：随机抽取验证存在性和内容匹配
5. **可重复性**：方法描述详细到第三方可重复

**严禁行为清单**
- 虚假数据、模拟数据冒充、示例数据支撑结论
- AI生成数据、选择性报告(p-hacking)、结果捏造
- 引用造假、方法混淆
